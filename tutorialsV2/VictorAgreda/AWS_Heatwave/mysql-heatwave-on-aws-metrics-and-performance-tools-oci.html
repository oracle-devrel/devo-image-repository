As you’re working with your data, you’ll want to check out the performance of MySQL HeatWave on AWS, and we provide a number of metrics for you to examine.

HeatWave Cluster Workspaces

Above, we see the Workspaces tab in the Console, having just run a query. While we see a memory snapshot, we can go deeper in the Performance tab.

Performance

Here’s the good stuff! Not only can you see performance per node, you can see the size of the dataset, the data dictionary, and if you click on Workload on the left side, you’ll see duration each step of the query took, and when queries have taken place.

Clicking back on Cluster and scrolling down, we can see metrics related to the VM itself, including memory and connection usage, CPU, and so on. If you’re keen on squeezing every drop of performance that you can out of HeatWave on AWS, we got you.

Getting there on OCI

However, HeatWave on OCI's Cloud Analytics platform is slightly different. You can find full documentation for using Oracle Analytics Cloud service with MySQL HeatWave here. For now, I'll just give a brief overview of how you get started:

We create an Analytics instance in our compartment.

Notice that as it starts up, the Analytics instance gives you an access URL -- that's because the Oracle Cloud Analytics is its own powerful platform including ML and other tools to help shape your solution to your needs. Also, you can set up Analytics so anyone in your organization can view data or even run some analytics without the concern of them having improper access.

Once in Analytics, you'll go to Connect to Your Data

Here we can see there are quite a few options! But we'll want humble but mighty MySQL.

Once we go to configure this connection, we see our usual inputs for authentication and connecting! After that, the metrics, etc. are pretty similar to what you see on the AWS side, with the addition of the Cloud Analytics tools.
